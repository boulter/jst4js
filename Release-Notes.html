Release Notes

Release 1.3

  • Added the Template.scripts() and Template.styles() methods for allowing partials to add dependencies on CSS and client Javascript paths for HTML templates.
  • Updated the Shell so it removes the bottom Text box, and behaves more like a shell you'd expect to see.
  • Refactored Template.collectContentFor() to handle multiple calls to collectContentFor() the resulting function will concatenate them together.

Release 1.2

  • Added support for treating instances of java.util.* as native Javascript types.
  • Added support for templates to declare the input parameters to their template. It simplifies all Templates both the top level template and partials you might call so they all behave the same way. All templates have formal parameters just like functions.
  • Also allow for layout definitions to be included within the template itself.
  • Updated Emailer to have support for specifying the username and password in spring configuration directly instead of through javamail properties.
